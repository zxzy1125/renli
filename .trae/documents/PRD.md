# 求职者转化话术生成工具 - 产品需求文档（PRD）

## 1. 产品概述

为人力代招公司打造的**公网部署团队版求职者转化辅助工具**，含 Web 应用 + Chrome 浏览器插件。

- **目标用户**：人力代招公司管理员、招聘顾问（员工）
- **部署方式**：公网部署（云端服务器 + 域名访问）
- **核心价值**：
  - 把"HR 凭经验临场组织话术"升级为"AI 起草 + 人工微调"
  - 通过回访计划避免员工"聊完就忘"，让每个求职者被持续触达
  - AI 深度分析求职者顾虑，给出能直接落地的应对策略
  - 撞单检测避免团队内部竞争
  - 浏览器插件一键读取 Boss/猎聘简历，免去复制粘贴

## 2. 用户角色与权限

### 2.1 角色定义

| 角色 | 核心权限 |
|------|---------|
| 管理员 | 全部数据可见、职位库录入、客户公司管理、撞单处理、员工管理、AI 配置、团队报表 |
| 员工 | 自己的简历/匹配/话术/跟进/转化，可查看团队共享的职位库和客户公司 |

### 2.2 数据权限矩阵

| 数据类型 | 录入权 | 可见范围 |
|---------|--------|---------|

| 客户公司 | 🔒 仅管理员 | 🟢 团队共享 |
| 求职者简历 | 🟢 所有员工 | 🔴 仅本人 + 管理员 |
| 匹配记录 | 🟢 所有员工 | 🔴 仅本人 + 管理员 |
| 话术 | 🟢 所有员工 | 🔴 仅本人 + 管理员 |
| 转化记录 | 🟢 所有员工 | 🔴 仅本人 + 管理员 |
| 跟进计划 | 🟢 所有员工 | 🔴 仅本人 + 管理员 |
| AI 分析报告 | 🟢 所有员工 | 🔴 仅本人 + 管理员 |
| 撞单记录 | 自动生成 | 🔴 仅管理员 |

## 3. 核心功能模块

1. **登录鉴权**：账号密码登录、JWT Token、RBAC 权限
2. **总览工作台**：转化漏斗、今日待回访、快捷入口、最近匹配
3. **职位库**（管理员录入，全员可见）：粘贴/上传 → AI 解析 → 列表/详情
4. **客户公司管理**（管理员）：客户公司 CRUD、关联职位
5. **简历库**（员工私有）：粘贴/上传/插件导入 → AI 解析 → 列表/详情 + 微信号/联系偏好 + 风险识别 + 共同点挖掘 + 人选备注
6. **匹配与话术生成**（核心）：选职位+选简历 → 匹配报告 + 18 条话术矩阵 → 半人工审核（话术遵守 BOSS 规范硬约束）
7. **跟进回访管理**（核心）：3 种计划类型 + 今日待回访 + AI 作战卡片 + AI 深度分析
8. **转化跟踪**：状态流转、时间轴、漏斗（按 BOSS 约面 4 步走：报备→确认函→确定回复→出发前提醒）
9. **撞单管理**（管理员）：撞单检测、撞单处理（归属/共享/误报）
10. **运营规范知识库**（内置 BOSS 规范）：员工可查、AI 调用时硬约束
11. **设置**：个人设置、团队管理、AI 配置、提示词配置、数据管理
12. **浏览器插件**：Chrome Extension，一键读取 Boss/猎聘简历页

### 3.1 BOSS 规范融入产品的三大能力

| 能力 | 实现方式 |
|------|---------|
| AI 话术硬约束 | 所有话术/回访/对话类 AI 调用，system prompt 顶部强制注入 BOSS 沟通禁区规则 |
| 风险求职者识别 | 简历解析时 AI 自动识别"风险画像"（外地求职者/质疑身份/套话等），打红标警告 |
| 标准沟通流程引导 | 话术场景对齐 BOSS 6 步流程：自我介绍→共同话题→居住地→语音铺垫→岗位沟通→约面 |

### 3.2 简历详情新增字段（基于 BOSS 规范）

| 字段 | 用途 |
|------|------|
| expectedCity（意向城市） | 用于风险识别（与岗位城市不一致=外地风险） |
| commonGrounds（共同点） | 4 类：校友/老乡/前公司/爱好，用于破冰话术 |
| riskWarning（风险警告） | AI 识别的风险求职者标记 + 原因列表 |
| remark（人选备注） | 自动生成"时间+姓名+年龄+学历+岗位方向"格式 |
| wechatId + hasWechat | 是否已加微信（员工手动维护） |
| contactPreference | 联系偏好（微信/电话/站内信） |

## 4. 核心流程

### 4.1 职位入库流程（管理员）

1. 管理员收到客户公司职位描述
2. 在"职位库"点击"新建职位"
3. 粘贴文本 或 上传文件
4. 点击"AI 解析" → 自动填充结构化字段
5. 管理员审核编辑 → 保存入库 → 全员可见

### 4.2 简历入库流程（员工）

**方式 A：复制粘贴/上传文件**
1. 求职者咨询，员工收到简历
2. 在"简历库"点击"新建简历"
3. 粘贴文本 或 上传文件
4. 点击"AI 解析" → 自动填充（含手机号脱敏）

**方式 B：浏览器插件一键导入（推荐）**
1. 员工在 Boss/猎聘打开求职者简历页
2. 点击浏览器插件图标
3. 插件自动读取页面简历内容
4. 点击"导入到工具" → 自动跳转工具新建简历页 + 自动填充
5. 点击"AI 解析"补全结构化字段

### 4.3 撞单检测流程

1. 员工保存简历时系统自动做撞单检测
2. 检测字段：手机号哈希、邮箱、姓名+现公司
3. 未撞单 → 正常入库
4. 撞单 → 弹窗提示，员工选择"仍保存"或"放弃"
5. 仍保存 → 入库 + 创建撞单记录 + 通知管理员
6. 管理员在撞单管理页处理：归属 A / 归属 B / 共享独立跟进 / 误报

### 4.4 匹配与话术生成流程

1. 员工在"匹配"页选 1 个职位 + 1 份简历
2. 点击"开始匹配" → AI 生成匹配报告
3. HR 确认匹配合理 → 点击"生成话术套件"
4. AI 一次性生成 18 条话术（3 渠道 × 6 场景）
5. HR 进入审核区：每条可"接受/编辑后接受/重新生成/放弃"
6. 接受后 → 一键复制 → 发送求职者
7. 匹配记录自动进入"转化跟踪"

### 4.5 跟进回访流程（核心）

1. 员工为简历创建回访计划（一次性/周期/自定义）
2. 系统按计划生成"今日待回访"提醒
3. 员工点击"开始回访" → AI 自动生成**作战卡片**：
   - 求职者深度画像
   - 本次回访目标
   - 预判顾虑 + 击穿策略
   - 3 套开场话术 + 心理学依据
   - 推荐提问
   - 转化概率预判
4. 员工参考卡片联系求职者
5. 沟通完录入回访记录
6. AI 自动生成**深度分析报告**：
   - 对话深度解析
   - 顾虑深度分析（fact/emotion/logic 三种击穿策略）
   - 跟进策略建议（P0/P1/P2）
   - 转化概率更新
   - 下次回访时间建议
7. 员工参考 AI 报告（无需审核，直接参考）
8. 可一键生成针对每个顾虑的应对话术（3 渠道）
9. 设置下次回访
10. 介绍了新职位 → 自动创建新匹配 → 自动生成新 18 条话术

### 4.6 转化跟踪流程

1. 匹配生成后自动创建转化记录，状态"咨询中"
2. 状态流转：咨询中 → 已邀面试 → 面试通过 → 已发offer → 已入职（或 已流失）
3. 每次状态变更自动写入时间轴
4. 总览页漏斗图实时统计

## 5. 话术矩阵设计

### 5.1 渠道（3个）

| 渠道 | 特点 |
|------|------|
| 微信文字 | 口语化、短句、可带emoji，≤200字 |
| 电话话术 | 结构化（开场→共鸣→说服→应对→收尾），500-800字 |
| 站内信 | 强钩子首句，200-400字 |

### 5.2 场景（6个）

| 场景 | 用途 |
|------|------|
| 触达开场白 | 首次接触，目标=让对方回复 |
| 职位介绍 | 让对方听完职位后产生兴趣 |
| 疑虑应对 | 针对匹配报告的 concerns 逐一击穿 |
| 面试邀约 | 推动接受面试 |
| 薪资沟通 | 基于 salaryAnalysis 谈薪 |
| Offer 促签 | 制造紧迫感推进签约 |

### 5.3 话术套件矩阵

3 渠道 × 6 场景 = **18 条话术**

## 6. AI 能力清单（9 套提示词）

| 序号 | 能力 | 触发时机 | 使用权限 | 输出 |
|------|------|---------|---------|------|
| 1 | 职位文件解析 | 管理员录入职位时粘贴/上传后点"AI 解析" | 管理员 | 结构化 JSON（解析+入库一体，员工看到的是结构化清晰版） |
| 2 | 简历文件解析 | 员工录入简历 | 员工 | 结构化 JSON（含脱敏） |
| 3 | 匹配分析 | 创建匹配 | 员工 | 评分+亮点+疑虑+薪资+预测 |
| 4 | 18 条话术生成 | 匹配后批量生成 | 员工 | 18 条 JSON |
| 5 | 回访前作战卡片 | 员工点"开始回访" | 员工 | 深度画像+顾虑+话术+策略 |
| 6 | 回访后深度分析 | 员工录入回访后 | 员工 | 顾虑解析+策略+下次建议 |
| 7 | 应对话术生成 | 处理具体顾虑 | 员工 | 3 渠道各 1 条话术 |
| 8 | 话术润色优化 | 员工编辑话术 | 员工 | 优化后话术+改动说明 |
| 9 | BOSS 岗位发布文案生成 | 职位详情页点"生成BOSS发布文案" | 🟢 全员 | 3 套文案（诱惑/神秘/专业） |

> **说明 1**：职位解析是**录入流程的环节**，不是独立功能。管理员粘贴客户发来的杂乱原文（常是表格），AI 解析成结构化字段后入库。员工在职位库看到的是**结构化清晰版**（职位名/薪资/职责/要求等字段化展示），原始文本作为 rawText 保留，可在详情页折叠区展开核对。

> **说明 2**：职位**录入**仅管理员（保存到职位库），BOSS 岗位发布文案生成全员可用（基于已入库职位反推发布文案，不修改职位数据）。

详细提示词见 [Prompts.md](file:///workspace/.trae/documents/Prompts.md)

## 7. 浏览器插件设计

### 7.1 支持平台

- Boss 直聘（Geek 端简历页）
- 猎聘（个人主页）

### 7.2 核心功能

| 功能 | 说明 |
|------|------|
| 一键读取简历 | 自动提取当前页面求职者信息（姓名/经验/技能/项目等） |
| 一键导入工具 | 跳转到工具新建简历页，自动填充原文 |
| 调用 AI 解析 | 在插件弹窗内直接调 AI 解析结构化字段 |
| 自动检测平台 | 根据当前 URL 自动启用对应解析规则 |

### 7.3 合规边界

- ✅ 员工手动点击触发（不自动爬取）
- ✅ 只读取当前页面（员工本来就能看到的）
- ✅ 不模拟发送消息（不替代用户操作）
- ✅ 不批量抓取
- ❌ 不做自动打招呼（合规风险）

### 7.4 技术方案

- Manifest V3
- Content Script 注入页面读取 DOM
- Background Service Worker 处理逻辑
- Popup 弹窗 UI
- 与主工具通过 API 通信（带 JWT Token）

## 8. 用户界面设计

### 8.1 设计风格

- **视觉风格**：专业商务风（Editorial Business），深墨绿 + 暖米白 + 赭石橙
- **字体**：标题思源宋体、正文思源黑体、数字 JetBrains Mono
- **按钮**：主按钮实心深墨绿，AI 按钮琥珀色渐变
- **布局**：左侧固定导航 + 右侧主内容
- **图标**：lucide-react，1.5px 描边

### 8.2 响应式

- 桌面优先（≥1280px 完整布局）
- 平板（768-1280px）：话术矩阵改列表
- 移动端：单栏堆叠

## 9. 数据存储

- **数据库**：SQLite（轻量，单文件，方便部署）
- **文件存储**：上传的职位/简历原文存数据库
- **AI 调用**：通过后端代理（隐藏 API Key）
- **会话**：JWT Token + Redis（可选，MVP 用内存）

## 10. 部署方案

### 10.1 服务器要求

- Linux 服务器（2核 4G 起步）
- Node.js 20+
- Nginx（反向代理 + 静态资源）
- SQLite 数据库文件

### 10.2 部署架构

```
公网用户
   ↓
Nginx（HTTPS + 域名）
   ↓
┌─────────────┬──────────────┐
│ 静态资源     │ API 反代      │
│ (前端 dist) │ → Node.js 后端│
└─────────────┴──────────────┘
                   ↓
              SQLite 文件
                   ↓
              AI 服务商 API
```

### 10.3 Docker 部署

提供 docker-compose.yml，一键启动：
- web 服务（前端静态 + 后端 API）
- 数据卷挂载 SQLite 文件
- 自动 HTTPS（Let's Encrypt）

## 11. 安全要求

- HTTPS 强制
- 密码 bcrypt 加密
- JWT Token 鉴权
- RBAC 权限校验
- 手机号脱敏存储 + 哈希检测撞单
- API Key 仅存后端，前端不可见
- SQL 参数化查询防注入
- 操作日志审计
